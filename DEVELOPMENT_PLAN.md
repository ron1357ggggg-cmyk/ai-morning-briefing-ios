# 開發計畫

## 第一階段

- 完成 SwiftUI 單頁晨報。
- 完成 HealthKit 授權與每日快照。
- 完成 7、30、90 日基準。
- 完成可測試的恢復分數與建議規則。
- 完成新聞 provider abstraction。
- 完成 08:30 本機通知。

## 第二階段

- 建立後端晨報 API。
- 串接具授權條款的新聞來源。
- 後端執行去重、影響力排序與繁中 AI 摘要。
- 使用 APNs 推送已完成的晨報。

## 第三階段

- 導入經審查的同齡研究資料集。
- 依年齡、性別、裝置與量測方法校準百分位。
- 增加長期趨勢與健力訓練紀錄。

## 技術限制

- iOS 本機通知可在 08:30 提醒，但不能保證 App 在該時刻前於背景完成網路工作。
- HealthKit 權限必須由使用者逐項授權，App 無法判斷使用者拒絕讀取某項資料。
- 睡眠資料可能來自多個來源，正式版需建立來源優先與去重規則。
- 健康建議必須標示非醫療用途。
- API key 不得放入 iOS bundle。
