# 測試規範

- 恢復分數需測試權重、狀態區間與 30 日差異。
- 新聞需測試 10 則配額、台灣至少 3 則、其他至少 7 則。
- RSS parser 測試必須使用人工 fixture，不依賴測試當下的外部網路。
- 每則新聞必須有可解析的 `sourceURL` 才可顯示「閱讀原文」。
- 健康與新聞需驗證單一來源失敗時，另一來源仍可完成更新。
- UI 手動檢查需涵蓋示範健康資料、即時新聞、離線備援與錯誤提示。
- 實機發佈前需重新驗證 HealthKit 授權、通知授權、安裝與啟動。
- `AppModel` 必須測試健康成功／新聞失敗，以及健康失敗／新聞成功的部分成功行為。
- 測試不得依賴真實 HealthKit、真實帳號、即時 RSS 內容或執行當下時間。
- 101 次重複測試用於發現狀態洩漏與偶發失敗，不代表形式化證明或零缺陷保證。
- 任一次重複測試失敗都視為失敗，不可只報告最後一次成功。
